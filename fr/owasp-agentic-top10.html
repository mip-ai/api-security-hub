OWASP Top 10 pour les applications agentiques (2026)

Les 10 risques de sécurité les plus critiques pour les systèmes d'agents IA et comment les atténuer.

Qu'est-ce que l'OWASP Top 10 pour les applications agentiques ?

Un classement des risques de sécurité les plus critiques spécifiques aux systèmes d'agents IA qui planifient de manière autonome, utilisent des outils et interagissent avec des services externes. L'édition 2026 aborde les menaces émergentes à mesure que l'IA agentique passe de la recherche aux déploiements en production.

1️⃣ ASI01 - Détournement des objectifs de l'agent

Critical

Vue d'ensemble

Un attaquant manipule les buts ou objectifs d'un agent par le biais d'entrées conçues à dessein, l'amenant à poursuivre des cibles non prévues. Contrairement à une simple injection de prompt, le détournement d'objectif peut persister à travers plusieurs étapes de planification, amenant l'agent à effectuer une série d'actions nuisibles de manière autonome.

Risque

Les attaquants peuvent rediriger des agents autonomes pour exfiltrer des données, modifier des configurations système ou effectuer des chaînes d'attaque en plusieurs étapes difficiles à détecter car l'agent semble fonctionner normalement.

Exemple de code vulnérable

Python ❌ Bad 
# Agent goal is derived directly from untrusted input
def run_agent(user_request: str) -> str:
    goal = f"Complete this task: {user_request}"
    plan = llm.plan(goal)
    for step in plan:
        execute(step)  # No validation of planned steps

Exemple de code sécurisé

Python ✅ Good 
import re

ALLOWED_GOALS = ["summarize", "search", "draft_email", "analyze_data"]

def sanitize_goal(user_request: str) -> str:
    # Strip injection patterns
    cleaned = re.sub(r'(?i)(ignore|override|new goal|forget).*', '', user_request)
    return cleaned.strip()

def run_agent(user_request: str) -> str:
    sanitized = sanitize_goal(user_request)
    goal = f"Complete this task: {sanitized}"
    plan = llm.plan(goal)

    # Validate each step against allowed actions
    for step in plan:
        if step.action not in ALLOWED_GOALS:
            raise ValueError(f"Disallowed action: {step.action}")
        if goal_drift_detected(step, sanitized):
            raise ValueError("Goal drift detected, aborting")
    for step in plan:
        execute(step)

Liste de contrôle des mesures d'atténuation

  • Assainir et valider toutes les entrées avant la formulation des objectifs
  • Mettre en œuvre la détection de dérive d'objectif à travers les étapes de planification
  • Maintenir une liste blanche d'objectifs autorisés pour l'agent
  • Enregistrer tous les changements d'objectif et déclencher des alertes en cas de modifications inattendues

2️⃣ ASI02 - Mauvaise utilisation et exploitation d'outils

Critical

Vue d'ensemble

Les agents ayant accès à des outils externes (API, systèmes de fichiers, bases de données, navigateurs web) peuvent être manipulés pour mal utiliser ces outils. Un accès non restreint aux outils permet aux attaquants d'effectuer des opérations non autorisées par le biais de l'agent.

Risque

Un agent avec un accès non restreint aux outils pourrait supprimer des fichiers, envoyer des requêtes API non autorisées, exfiltrer des données via la navigation web ou modifier des configurations système critiques.

Exemple de code vulnérable

Python ❌ Bad 
# Agent can call any tool without restrictions
def agent_execute(tool_name: str, params: dict):
    tool = tools_registry.get(tool_name)
    return tool(**params)  # No validation or approval

Exemple de code sécurisé

Python ✅ Good 
TOOL_ALLOWLIST = {
    "web_search": {"max_calls": 10, "approval": False},
    "send_email": {"max_calls": 1, "approval": True},
    "file_write": {"max_calls": 5, "approval": True},
}

def agent_execute(tool_name: str, params: dict, session) -> str:
    if tool_name not in TOOL_ALLOWLIST:
        return "Error: Tool not permitted"

    config = TOOL_ALLOWLIST[tool_name]
    if session.tool_calls[tool_name] >= config["max_calls"]:
        return "Error: Tool call limit exceeded"

    if config["approval"]:
        if not request_human_approval(tool_name, params):
            return "Action denied by user"

    session.tool_calls[tool_name] += 1
    return tools_registry[tool_name](**params)

Liste de contrôle des mesures d'atténuation

  • Maintenir une liste blanche stricte des outils que l'agent peut invoquer
  • Exiger une approbation humaine pour les opérations d'outils à fort impact
  • Appliquer des limites de taux par session et par outil
  • Valider les paramètres des outils par rapport aux schémas attendus avant l'exécution

3️⃣ ASI03 - Abus d'identité et de privilèges

Critical

Vue d'ensemble

Les agents héritent souvent de l'identité et des permissions de l'utilisateur ou du compte de service qui les a lancés. Cet héritage excessif de privilèges permet aux agents d'effectuer des actions au-delà de ce qui est nécessaire, créant une large surface d'attaque si l'agent est compromis.

Risque

Un agent compromis fonctionnant avec des identifiants administrateur peut accéder à tous les systèmes, modifier les permissions et escalader les privilèges dans toute l'organisation.

Exemple de code vulnérable

Python ❌ Bad 
# Agent inherits full user credentials
def create_agent(user_session):
    agent = Agent(
        credentials=user_session.full_credentials,  # All permissions!
        scope="*",
    )
    return agent

Exemple de code sécurisé

Python ✅ Good 
def create_agent(user_session, task_type: str):
    # Issue scoped, short-lived credentials for the agent
    scoped_token = auth.create_scoped_token(
        parent_token=user_session.token,
        scopes=TASK_SCOPES[task_type],  # Minimal required permissions
        ttl_minutes=30,
        max_actions=50,
    )

    agent = Agent(
        credentials=scoped_token,
        scope=TASK_SCOPES[task_type],
        audit_log=True,
    )
    return agent

TASK_SCOPES = {
    "summarize": ["read:documents"],
    "draft_email": ["read:contacts", "draft:email"],
    "analyze": ["read:data", "write:reports"],
}

Liste de contrôle des mesures d'atténuation

  • Émettre des identifiants à portée limitée et de courte durée spécifiques à la tâche de l'agent
  • Ne jamais transmettre les identifiants complets de l'utilisateur ou de l'administrateur aux agents
  • Mettre en œuvre des limites de permissions par tâche avec des limites d'action maximales
  • Auditer toutes les opérations privilégiées effectuées par les agents

4️⃣ ASI04 - Vulnérabilités de la chaîne d'approvisionnement agentique

High

Vue d'ensemble

Les systèmes d'agents s'appuient sur des plugins tiers, des intégrations d'outils et des frameworks d'agents partagés. Des composants compromis ou malveillants dans la chaîne d'approvisionnement des agents peuvent introduire des portes dérobées, des canaux d'exfiltration de données ou des capacités non autorisées.

Python ❌ Bad 
# Loading plugins without verification
def load_plugin(plugin_url: str):
    code = requests.get(plugin_url).text
    exec(code)  # Arbitrary code execution!
Python ✅ Good 
import hashlib, importlib

TRUSTED_PLUGINS = {
    "search_plugin": "sha256:a1b2c3...",
    "email_plugin": "sha256:d4e5f6...",
}

def load_plugin(plugin_name: str) -> None:
    if plugin_name not in TRUSTED_PLUGINS:
        raise ValueError(f"Untrusted plugin: {plugin_name}")

    module = importlib.import_module(f"plugins.{plugin_name}")
    actual_hash = compute_hash(module.__file__)

    if actual_hash != TRUSTED_PLUGINS[plugin_name]:
        raise ValueError("Plugin integrity check failed")

    module.init(sandbox=True)

Liste de contrôle des mesures d'atténuation

  • Vérifier l'intégrité des plugins avec des signatures cryptographiques avant le chargement
  • Maintenir un registre organisé de plugins et d'outils d'agents approuvés
  • Exécuter les plugins tiers dans des environnements sandbox avec des permissions limitées
  • Mettre en œuvre le suivi SBOM pour toutes les dépendances du framework d'agents

5️⃣ ASI05 - Exécution de code inattendue

Critical

Vue d'ensemble

Les agents qui peuvent générer et exécuter du code (par exemple, agents d'analyse de données, assistants de codage) peuvent être trompés pour exécuter du code malveillant. Sans sandbox approprié, cela peut conduire à la compromise du système, au vol de données ou à des mouvements latéraux.

Risque

Les agents utilisant eval() ou exec() sur du code généré peuvent être exploités pour l'exécution de code à distance, permettant aux attaquants d'obtenir un accès complet au système.

Exemple de code vulnérable

Python ❌ Bad 
# Agent executes generated code directly
def code_agent(task: str) -> str:
    code = llm.generate_code(task)
    result = eval(code)  # Dangerous!
    return str(result)

Exemple de code sécurisé

Python ✅ Good 
import subprocess, tempfile, os

BLOCKED_MODULES = ["os", "subprocess", "socket", "shutil"]

def code_agent(task: str) -> str:
    code = llm.generate_code(task)

    # Static analysis: block dangerous imports
    for mod in BLOCKED_MODULES:
        if f"import {mod}" in code or f"from {mod}" in code:
            raise ValueError(f"Blocked import: {mod}")

    # Execute in sandboxed container with resource limits
    result = sandbox.run(
        code=code,
        timeout=30,
        memory_mb=256,
        network=False,
        read_only_fs=True,
    )
    return result.output

Liste de contrôle des mesures d'atténuation

  • Ne jamais utiliser eval() ou exec() sur du code généré par l'agent dans l'environnement hôte
  • Exécuter le code généré dans des conteneurs sandbox avec des limites de ressources
  • Effectuer une analyse statique pour bloquer les importations dangereuses et les appels système
  • Désactiver l'accès réseau et appliquer des systèmes de fichiers en lecture seule dans les environnements d'exécution de code

6️⃣ ASI06 - Empoisonnement de la mémoire et du contexte

High

Vue d'ensemble

Les agents qui maintiennent une mémoire persistante (RAG, historique de conversation, préférences apprises) sont vulnérables à l'empoisonnement de la mémoire. Les attaquants injectent du contenu malveillant dans la base de connaissances de l'agent, l'amenant à produire des sorties compromises lors d'interactions futures.

Python ❌ Bad 
# Agent stores all interactions without validation
def store_memory(agent_id: str, interaction: str):
    memory_db.insert(agent_id, interaction)  # No filtering
Python ✅ Good 
def store_memory(agent_id: str, interaction: str, source: str):
    # Validate content before storing
    if contains_injection_patterns(interaction):
        log.warning(f"Blocked poisoned memory: {agent_id}")
        return

    memory_db.insert(
        agent_id=agent_id,
        content=interaction,
        source=source,
        provenance=compute_provenance(source),
        timestamp=now(),
        ttl_days=30,  # Auto-expire old memories
    )

def retrieve_memory(agent_id: str, query: str) -> list:
    results = memory_db.search(agent_id, query)
    # Filter by provenance score
    return [r for r in results if r.provenance_score > 0.8]

Liste de contrôle des mesures d'atténuation

  • Valider et assainir toutes les entrées avant de les stocker dans la mémoire de l'agent
  • Suivre la provenance de toutes les entrées de mémoire et filtrer par score de confiance
  • Mettre en œuvre le TTL (durée de vie) pour les entrées de mémoire afin de limiter la persistance de l'empoisonnement
  • Surveiller les modèles anormaux dans la mémoire stockée et activer la restauration

7️⃣ ASI07 - Communication inter-agents non sécurisée

High

Vue d'ensemble

Les systèmes multi-agents où les agents communiquent entre eux sont vulnérables à la falsification de messages, à l'usurpation d'identité et à l'écoute clandestine. Sans authentification appropriée et vérifications d'intégrité, un agent compromis peut injecter des instructions malveillantes dans le réseau d'agents.

Python ❌ Bad 
# Agents communicate via plain text messages
def send_to_agent(target: str, message: str):
    channel.send(target, message)  # No auth, no signing
Python ✅ Good 
import hmac, json, time

def send_to_agent(target: str, message: str, sender_key: bytes):
    payload = {
        "content": message,
        "sender": agent_id,
        "target": target,
        "timestamp": time.time(),
        "nonce": os.urandom(16).hex(),
    }
    signature = hmac.new(
        sender_key, json.dumps(payload).encode(), "sha256"
    ).hexdigest()
    payload["signature"] = signature

    encrypted = encrypt(json.dumps(payload), target_public_key)
    channel.send(target, encrypted)

def receive_message(data: bytes, private_key) -> dict:
    payload = json.loads(decrypt(data, private_key))
    if not verify_signature(payload):
        raise ValueError("Invalid message signature")
    if is_replay(payload["nonce"]):
        raise ValueError("Replay attack detected")
    return payload

Liste de contrôle des mesures d'atténuation

  • Signer tous les messages inter-agents avec HMAC ou des signatures numériques
  • Chiffrer les canaux de communication inter-agents de bout en bout
  • Mettre en œuvre la prévention des attaques par rejeu basée sur les nonces
  • Authentifier l'identité de l'agent avant d'accepter les messages

8️⃣ ASI08 - Défaillances en cascade

High

Vue d'ensemble

Dans les flux de travail multi-agents ou multi-étapes, une erreur ou une action malveillante dans un agent peut se propager dans le système, provoquant des défaillances en cascade. Sans limites d'erreur appropriées, une seule étape compromise peut corrompre l'ensemble du pipeline.

Python ❌ Bad 
# Errors propagate without boundaries
def pipeline(data):
    result1 = agent_a.process(data)
    result2 = agent_b.process(result1)  # If agent_a fails or is poisoned...
    result3 = agent_c.process(result2)  # ...error cascades to all
    return result3
Python ✅ Good 
from circuitbreaker import circuit

class AgentPipeline:
    def __init__(self):
        self.circuit_breakers = {}

    @circuit(failure_threshold=3, recovery_timeout=60)
    def safe_execute(self, agent, data):
        result = agent.process(data)
        if not validate_output(result):
            raise ValueError("Output validation failed")
        return result

    def pipeline(self, data):
        try:
            r1 = self.safe_execute(agent_a, data)
        except Exception:
            r1 = fallback_a(data)

        try:
            r2 = self.safe_execute(agent_b, r1)
        except Exception:
            r2 = fallback_b(r1)

        return r2

Liste de contrôle des mesures d'atténuation

  • Mettre en œuvre des disjoncteurs entre les étapes du pipeline d'agents
  • Valider les sorties à chaque étape du pipeline avant de passer à la suivante
  • Fournir des gestionnaires de secours pour chaque agent critique dans le flux de travail
  • Définir des délais d'expiration et des limites de réessai pour éviter les boucles infinies dans les chaînes d'agents

9️⃣ ASI09 - Exploitation de la confiance humain-agent

Medium

Vue d'ensemble

Les utilisateurs peuvent accorder une confiance excessive aux sorties des agents et approuver des actions sans examen adéquat. Les agents qui présentent des recommandations avec une grande confiance mais un fondement insuffisant peuvent amener les utilisateurs à prendre des décisions préjudiciables. Les attaquants peuvent exploiter cette relation de confiance.

Python ❌ Bad 
# Agent requests approval without context
def request_action(action: str):
    # "Deploy to production?" - user clicks Yes without review
    return ui.confirm(f"Execute: {action}?")
Python ✅ Good 
def request_action(action: str, context: dict) -> bool:
    confidence = context.get("confidence", 0.0)
    risk_level = assess_risk(action)

    approval_request = {
        "action": action,
        "confidence": f"{confidence:.0%}",
        "risk_level": risk_level,
        "reasoning": context["reasoning"],
        "affected_systems": context["systems"],
        "reversible": context.get("reversible", False),
    }

    # Force detailed review for high-risk or low-confidence
    if risk_level == "high" or confidence < 0.8:
        return ui.detailed_review(approval_request)

    return ui.confirm(approval_request)

Liste de contrôle des mesures d'atténuation

  • Afficher les scores de confiance et le raisonnement pour toutes les recommandations de l'agent
  • Exiger un examen détaillé pour les actions à haut risque ou à faible confiance
  • Afficher les systèmes affectés et l'état de réversibilité dans les dialogues d'approbation
  • Mettre en œuvre la confiance progressive : commencer avec plus de supervision, réduire à mesure que l'agent se révèle fiable

🔟 ASI10 - Agents voyous

High

Vue d'ensemble

Les agents peuvent s'écarter de leur objectif prévu en raison d'un désalignement des objectifs, d'une manipulation adversariale ou de comportements émergents. Les agents voyous peuvent poursuivre des objectifs qui entrent en conflit avec les buts organisationnels, accumuler des ressources ou résister aux tentatives d'arrêt.

Python ❌ Bad 
# Agent runs without monitoring or kill switch
def run_agent(task):
    while True:
        agent.step()  # No termination condition
Python ✅ Good 
class MonitoredAgent:
    def __init__(self, agent, max_steps=100):
        self.agent = agent
        self.max_steps = max_steps
        self.step_count = 0
        self.behavior_log = []

    def run(self):
        while self.step_count < self.max_steps:
            action = self.agent.next_action()

            # Check for rogue behavior
            if self.is_off_task(action):
                log.alert(f"Rogue behavior: {action}")
                self.shutdown()
                return

            # Check guardrails
            if not guardrails.check(action):
                log.warning(f"Guardrail violation: {action}")
                continue

            self.agent.execute(action)
            self.step_count += 1
            self.behavior_log.append(action)

    def shutdown(self):
        self.agent.stop()
        revoke_credentials(self.agent.id)
        notify_admin(self.behavior_log)

Liste de contrôle des mesures d'atténuation

  • Mettre en œuvre la surveillance du comportement avec détection d'anomalies pour tous les agents
  • Définir des limites d'étapes maximales et appliquer la terminaison automatique
  • Déployer des garde-fous qui vérifient chaque action par rapport à la politique avant l'exécution
  • Maintenir un bouton d'arrêt d'urgence qui révoque les identifiants et arrête l'exécution de l'agent

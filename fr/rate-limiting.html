⏱ Limitation du débit

La limitation de débit est un mécanisme de défense fondamental pour protéger votre système contre les abus d'API, les attaques DoS et le scraping.

Comparaison d'algorithmes

Implémentation de la limitation de débit dans Express.js

JavaScript (Express)Limitation de débit multicouche
const rateLimit = require('express-rate-limit');
const RedisStore = require('rate-limit-redis');

// Limite globale : appliquée à toutes les API
const globalLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100,
  standardHeaders: true,    // Renvoie les en-têtes RateLimit-*
  legacyHeaders: false,
  message: { error: 'Too many requests, please try again later.' },
});

// Pour les points de terminaison d'authentification : limites plus strictes
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 5,  // Tentatives de connexion limitées à 5 toutes les 15 minutes
  skipSuccessfulRequests: true, // Les requêtes réussies ne sont pas comptées
});

// Pour les environnements distribués : backend Redis
const distributedLimiter = rateLimit({
  store: new RedisStore({ sendCommand: (...args) => redisClient.sendCommand(args) }),
  windowMs: 60 * 1000,
  max: 30,
});

app.use(globalLimiter);
app.use('/api/auth', authLimiter);

Touches de limitation de débit

En appliquant des limites de débit à l'aide de clés composites telles qu'une clé API, un ID utilisateur ou un point de terminaison en plus de l'adresse IP, vous pouvez minimiser l'impact sur les utilisateurs légitimes.

🔍 Validation des entrées

Toutes les entrées de l'API peuvent contenir des données malveillantes. Validez toujours côté serveur.

Principes de validation

Approche de liste verte

Définissez une liste de valeurs autorisées. Plus sûr qu'une liste de refus (liste de blocage). Peut gérer de nouveaux modèles d’attaque.

Type, longueur et portée

Vérifiez strictement les types de données, le nombre maximum de caractères et les limites numériques supérieures/inférieures. Une défense fondamentale contre les débordements de tampon.

Désinfecter ou rejeter

Rejeter une entrée invalide est plus sûr que de la « corriger ». La désinfection peut produire des transformations inattendues.

Validation avec le schéma JSON

JavaScript (Express + Ajv)Validation du schéma
const Ajv = require('ajv');
const addFormats = require('ajv-formats');

const ajv = new Ajv({ allErrors: true, removeAdditional: true });
addFormats(ajv);

// Schéma pour l'API de création d'utilisateurs
const createUserSchema = {
  type: 'object',
  required: ['name', 'email'],
  additionalProperties: false,
  properties: {
    name: {
      type: 'string',
      minLength: 1,
      maxLength: 100,
      pattern: '^[a-zA-Z0-9\\s\\-]+$', // Restreindre aux caractères autorisés
    },
    email: {
      type: 'string',
      format: 'email',
      maxLength: 254,
    },
    age: {
      type: 'integer',
      minimum: 0,
      maximum: 150,
    },
  },
};

// Middleware de validation
function validateBody(schema) {
  const validate = ajv.compile(schema);
  return (req, res, next) => {
    if (!validate(req.body)) {
      return res.status(400).json({
        error: 'Validation failed',
        details: validate.errors,
      });
    }
    next();
  };
}

app.post('/api/users', validateBody(createUserSchema), createUser);

🌐 CORS (Partage de ressources inter-origines)

CORS est un mécanisme qui contrôle la politique de même origine du navigateur. Une mauvaise configuration peut entraîner de graves risques de sécurité.

Configuration dangereuse

Access-Control-Allow-Origin : * et Access-Control-Allow-Credentials : true ne peuvent pas être utilisés ensemble. Restreindre l’utilisation des caractères génériques aux API publiques uniquement.

Configuration CORS sécurisée

JavaScript (Express)Configuration CORS
const cors = require('cors');

// Spécifie explicitement les origines autorisées
const allowedOrigins = [
  'https://app.example.com',
  'https://admin.example.com',
];

app.use(cors({
  origin(origin, callback) {
    // Autoriser la communication de serveur à serveur (pas d'origine)
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-API-Key'],
  credentials: true,
  maxAge: 86400,  // Cache de contrôle en amont : 24 heures
}));

Liste de contrôle CORS

  • N'utilisez pas de caractère générique (*) pour Access-Control-Allow-Origin (pour les API nécessitant une authentification)
  • Access-Control-Allow-Methods doit inclure uniquement les méthodes requises
  • Access-Control-Allow-Headers doit inclure uniquement les en-têtes requis
  • Mettre en cache les requêtes de contrôle en amont avec Access-Control-Max-Age
  • Restreindre les en-têtes de réponse exposés avec Access-Control-Expose-Headers
  • Ne reflète jamais l’origine de la demande directement dans la réponse

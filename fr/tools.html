Outils de sécurité des API

Présentation d'outils utiles pour l'évaluation, les tests et la surveillance des vulnérabilités des API.

🔎 Outils de tests dynamiques de sécurité des applications (DAST)

Outils qui envoient des requêtes aux API en cours d'exécution pour détecter les vulnérabilités.

Burp Suite Édition commerciale/gratuite disponible

L'outil standard de l'industrie pour les tests de sécurité des applications Web et des API. Capacités d’interception et d’analyse basées sur un proxy.

  • Intercepter et modifier les requêtes/réponses HTTP
  • Analyse automatisée (Injection SQL, XSS, etc.)
  • Intrus : fuzzing des paramètres
  • Répéteur : relecture et vérification manuelle des demandes
  • Écosystème d'extension riche (BApp Store)
Conseils pour les tests d'API

Importez une définition OpenAPI/Swagger pour explorer et tester automatiquement tous les points de terminaison.

OWASP ZAP Gratuit / OSS

Un outil de test de sécurité open source développé par OWASP. Facile à intégrer dans les pipelines CI/CD.

  • Analyse active / Analyse passive
  • Importer des définitions d'API (OpenAPI, GraphQL)
  • Intégration CI/CD (GitHub Actions, Jenkins)
  • Exécution sans tête avec prise en charge de Docker
  • Tests personnalisés via des scripts
BashAnalyse API avec ZAP Docker
docker run -t ghcr.io/zaproxy/zaproxy:stable zap-api-scan.py \
  -t https://api.example.com/openapi.json \
  -f openapi \
  -r report.html

🧪 Outils de test et de développement d'API

Facteur Gratuit / Commercial

L'outil incontournable pour le développement et les tests d'API. Également utile pour les tests de sécurité.

  • Organisez des cas de test avec la fonctionnalité Collections
  • Basculer entre la préparation et la production avec des variables d'environnement
  • Acquérir automatiquement des jetons avec des scripts de pré-demande
  • Automatisez la validation des réponses dans l'onglet Tests
  • Exécuter à partir de CI/CD avec Newman (CLI)
JavaScriptExemple de script de test Postman
// Validation du code d'état
pm.test("Status is 200", () => {
  pm.response.to.have.status(200);
});

// Validation de l'en-tête de sécurité
pm.test("Security headers present", () => {
  pm.response.to.have.header("X-Content-Type-Options");
  pm.response.to.have.header("Strict-Transport-Security");
});

// Vérification de l'exposition des données sensibles
pm.test("No sensitive data exposed", () => {
  const body = pm.response.json();
  pm.expect(body).to.not.have.property("password");
  pm.expect(body).to.not.have.property("password_hash");
  pm.expect(body).to.not.have.property("secret");
});

curl + jq Gratuit / CLI

Test rapide de l'API depuis la ligne de commande. Facile à scripter et à automatiser.

  • Disponible dans n’importe quel environnement
  • Automatisez avec des scripts shell
  • Analyser les réponses JSON avec jq
  • Inspection détaillée des en-têtes HTTP
BashVérification de l'en-tête de sécurité
# Vérifiez les en-têtes de réponse
curl -s -D - -o /dev/null https://api.example.com/health

# Tentative d'accès sans authentification
curl -s -w "%{http_code}" https://api.example.com/api/users

# Test BOLA (accès à la ressource d'un autre utilisateur)
curl -s -H "Authorization: Bearer $TOKEN" \
  https://api.example.com/api/users/OTHER_USER_ID

📊 Analyse statique (SAST) et analyse de la composition logicielle (SCA)

Semgrep Gratuit / OSS

Un outil d'analyse statique basé sur des modèles. Facile à créer des règles personnalisées.

  • Prise en charge multilingue (JS, Python, Go, Java, etc.)
  • Ensembles de règles de sécurité intégrés
  • Intégration CI/CD facile
  • Définir des règles personnalisées dans YAML

Snyk Niveau gratuit disponible

Analyse des vulnérabilités des dépendances et analyse du code.

  • Analyse des dépendances pour npm, pip, Maven, etc.
  • Détection automatique des CVE connus
  • Générer automatiquement des PR de correctifs
  • Analyse d'images de conteneurs

audit npm / Trivy Gratuit

Vérification des vulnérabilités pour les gestionnaires de packages et les conteneurs.

  • audit npm : packages Node.js
  • audit pip : packages Python
  • Trivy : Conteneurs, IaC et systèmes de fichiers
  • Idéal pour le gate CI/CD
BashVérification de la vulnérabilité des dépendances
# Vérifier les dépendances de Node.js
npm audit --production

# Analyse de sécurité avec Semgrep
semgrep --config=p/security-audit ./src

# Analyse de conteneurs avec Trivy
trivy image --severity HIGH,CRITICAL myapp:latest

📝 Gestion des spécifications et de la documentation des API

Swagger / OpenAPI Spécification standard

Définissez les spécifications de l'API dans un format lisible par machine. Également utilisé comme entrée pour les tests de sécurité.

  • Définition complète des points finaux, des paramètres et des réponses
  • Schémas de sécurité explicites (OAuth, clé API, etc.)
  • Importer dans ZAP/Burp pour automatiser les tests
  • Génération de documentation interactive avec l'interface utilisateur Swagger

Spectral Gratuit / OSS

Un outil linter pour les spécifications OpenAPI. Vérifie le respect des bonnes pratiques de sécurité.

  • Vérifier la présence de définitions de schéma de sécurité
  • Détecter les points de terminaison non authentifiés
  • Appliquer des politiques internes avec des règles personnalisées
  • Porte de qualité pour les spécifications API dans CI/CD

📡 Outils de surveillance et de journalisation

Passerelle API

Kong, AWS API Gateway, Apigee et autres.

  • Authentification et autorisation centralisées
  • Application de la limitation du débit
  • Demander l'agrégation des journaux
  • Alertes de détection d'anomalies

WAF

AWS WAF, Cloudflare, ModSecurity et autres.

  • Bloquer les modèles d'attaque tels que SQLi/XSS
  • Contrôle d'accès basé sur GeoIP
  • Atténuation des robots
  • Définitions de règles personnalisées

SIEM / Analyse des journaux

ELK Stack, Splunk, Datadog et autres.

  • Agrégation et visualisation des journaux API
  • Détection de modèles anormaux
  • Prise en charge de la réponse aux incidents
  • Audit de conformité

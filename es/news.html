🔥 Incidentes importantes

Critical 29 de enero de 2025 Exposición de datos

DeepSeek expone más de 1 millón de registros a través de una base de datos no autenticada: historial de chat y claves API filtrados

Wiz Research descubrió que la base de datos ClickHouse de DeepSeek era accesible públicamente sin autenticación en dos subdominios. Más de 1 millón de entradas de registro contenían historial de chat en texto plano, claves secretas de API y metadatos operativos de backend. Los atacantes podrían robar contraseñas y archivos locales directamente desde el servidor. Tras una divulgación responsable, DeepSeek aseguró inmediatamente la base de datos. El 27 de enero también se produjo un ataque DDoS a gran escala contra la API y la interfaz de chat web.

Fuente: Blog de Wiz

Critical 10 de mayo de 2024 Violación de datos

49 millones de registros de clientes robados a través del abuso de API del portal de socios de Dell

Un atacante registró una cuenta de socio falsa en el portal de socios de Dell y obtuvo acceso en 48 horas. La API del portal no tenía controles de autorización ni limitación de velocidad. Durante tres semanas, el atacante envió aproximadamente 5000 solicitudes por minuto y robó 49 millones de registros que contenían nombres, direcciones, etiquetas de servicio e información de pedidos. Los datos se vendieron en un foro de piratería.

Fuente: BleepingComputer

High 16 de enero de 2024 Violación de datos

15 millones de datos de usuarios de Trello filtrados a través de API Scraping

Un atacante aprovechó la API REST desprotegida de Trello, que permitía consultas no autenticadas por dirección de correo electrónico. Al enviar direcciones de correo electrónico de bases de datos previamente violadas, el atacante extrajo más de 15 millones de perfiles de usuario (correos electrónicos, nombres completos, nombres de usuario, información del foro) y los publicó en un foro de piratería. Posteriormente, Atlassian restringió el punto final.

Fuente: BleepingComputer

High julio 2025 Fuga de clave API

Empleado de DOGE publica clave API privada xAI en GitHub: acceso a más de 52 LLM internos expuestos

Un empleado del Departamento de Eficiencia Gubernamental de EE. UU. (DOGE) envió un script (agent.py) que contiene la clave API privada de xAI a un repositorio público de GitHub. La clave proporcionaba acceso ilimitado a más de 52 modelos LLM, incluido el "grok 4-0709", que se había creado apenas cuatro días antes. GitGuardian detectó la fuga, pero la clave no fue revocada de inmediato. También se produjo una filtración similar en mayo de 2025.

Fuente: Krebs sobre seguridad

🛡 CVE notables

Active Exploitation Marzo de 2025 (se observa explotación activa) SSRF

CVE-2024-27564: Más de 10 000 intentos de ataque a la vulnerabilidad ChatGPT SSRF

Una vulnerabilidad de falsificación de solicitudes del lado del servidor (SSRF) en el componente pictureproxy.php de ChatGPT. Un atacante no autenticado podría inyectar URL manipuladas para obligar al servidor a ejecutar solicitudes internas no autorizadas. Se registraron más de 10.000 intentos de ataque desde una única dirección IP, y el 33% de los objetivos eran organizaciones estadounidenses (sectores financiero, gubernamental y sanitario). La vulnerabilidad es particularmente peligrosa porque puede explotarse a escala sin autenticación.

Fuente: SecurityWeek

Lección aprendida: SSRF es una grave amenaza para las API

CVE-2024-27564 es un ejemplo de libro de texto de OWASP API7 (SSRF). Las API que aceptan entradas de URL de fuentes externas requieren una validación estricta para evitar el acceso a redes internas. Consulte OWASP Top 10 → API7.

📊 Informes y tendencias de la industria

Los CVE relacionados con la IA aumentan un 1.025% año tras año

Según el informe API ThreatStats 2025 de Wallarm, en 2024 se registraron 439 CVE relacionados con la IA, un aumento interanual del 1.025%. Más del 50% de las entradas del catálogo de vulnerabilidades explotadas conocidas (KEV) de CISA están relacionadas con API. Las fallas de control de acceso aumentaron un 40 % en general y las fallas de autorización críticas aumentaron un 36 %.

Wallarm 2025

El 99% de las organizaciones experimentaron problemas de seguridad de API

Según el informe del primer trimestre de 2025 de Salt Security, el 99% de las organizaciones encuestadas experimentaron al menos un problema de seguridad de API en los últimos 12 meses. Los principales riesgos fueron los ataques de inyección y BOLA (autorización de nivel de objeto roto), que representaron más de un tercio de todos los incidentes. El 95% de los ataques a API se originaron en sesiones autenticadas.

Salt Security Q1 2025

Se encontraron 30.000 espacios de trabajo de Postman exponiendo claves API

Los investigadores de seguridad descubrieron 30.000 espacios de trabajo de Postman expuestos públicamente sin controles de seguridad. Contenían claves API activas, tokens de acceso y cargas útiles confidenciales, muchas de las cuales proporcionaban acceso directo a entornos de producción. Este caso destaca el riesgo de las credenciales de "API oculta" en las herramientas de colaboración de desarrolladores.

Wallarm 2025

📈 Tendencias de seguridad de API 2024-2025

1. Rápido aumento de las vulnerabilidades de las API AI/LLM

Las vulnerabilidades en API basadas en IA, como ChatGPT y DeepSeek, están aumentando. La inyección rápida, SSRF y la exposición de datos son las principales amenazas. Las medidas de seguridad no van a la par de la rápida adopción de la IA.

2. La autorización incumplida sigue siendo la principal amenaza

BOLA (autorización a nivel de objeto roto) sigue siendo la vulnerabilidad más frecuente. Como se vio en los casos de Dell y Trello, la falta de controles de autorización básicos conduce a violaciones de datos a gran escala.

3. Ataques de sesiones autenticadas

El 95% de los ataques ocurren después de pasar la autenticación. Los atacantes con tokens de acceso legítimos aprovechan las fallas de autorización para moverse lateralmente, lo que la convierte en la técnica de ataque dominante.

4. Fugas de credenciales a través de herramientas de desarrollo

Las filtraciones de claves API de herramientas de desarrollo como GitHub y Postman se han convertido en un problema crítico. La adopción de herramientas de detección automatizadas (por ejemplo, GitGuardian) y prácticas rigurosas de gestión de secretos son esenciales.

✅ Acciones a realizar ahora

Contramedidas aprendidas de estos incidentes
  • Aplicar comprobaciones de autorización: implementar autorización a nivel de objeto en todos los puntos finales de API (lección de Dell/Trello)
  • Implementar limitación de velocidad: establezca límites de velocidad adecuados por terminal (Dell no tenía límites y permitía 5000 solicitudes por minuto).
  • Automatizar el análisis de secretos: detecta automáticamente fugas de credenciales de repositorios/espacios de trabajo públicos en GitHub, Postman, etc.
  • Implementar contramedidas SSRF: validar las URL proporcionadas por el usuario y bloquear rangos de direcciones IP privadas
  • Auditar la configuración de exposición de la base de datos: compruebe periódicamente que las interfaces de administración no queden expuestas involuntariamente (lección de DeepSeek)
  • Evaluación de seguridad para terminales AI/LLM: realice evaluaciones de riesgos para una rápida inyección y fuga de datos
  • Comprobaciones de vulnerabilidad de dependencia: integre npm audit/Snyk en CI/CD para detectar automáticamente CVE conocidos

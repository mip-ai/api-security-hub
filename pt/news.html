Critical 29 de janeiro de 2025 Exposição de dados

DeepSeek expõe mais de 1 milhão de registros por meio de banco de dados não autenticado - histórico de bate-papo e chaves de API vazadas

A Wiz Research descobriu que o banco de dados ClickHouse da DeepSeek era acessível publicamente sem autenticação em dois subdomínios. Mais de 1 milhão de entradas de log continham histórico de bate-papo em texto simples, chaves secretas de API e metadados operacionais de back-end. Os invasores podem roubar senhas e arquivos locais diretamente do servidor. Após a divulgação responsável, a DeepSeek protegeu imediatamente o banco de dados. Um ataque DDoS em grande escala contra a API e a interface de chat na web também ocorreu em 27 de janeiro.

Fonte: Wiz Blog

Critical 10 de maio de 2024 Violação de dados

49 milhões de registros de clientes roubados por meio de abuso de API do Dell Partner Portal

Um invasor registrou uma conta de parceiro falsa no portal de parceiros da Dell e obteve acesso em 48 horas. A API do portal não tinha verificações de autorização ou limitação de taxa. Durante três semanas, o invasor enviou aproximadamente 5.000 solicitações por minuto, roubando 49 milhões de registros contendo nomes, endereços, etiquetas de serviço e informações de pedidos. Os dados foram vendidos em um fórum de hackers.

Fonte: BleepingComputer

High 16 de janeiro de 2024 Violação de dados

Dados de 15 milhões de usuários do Trello vazados por meio de API Scraping

Um invasor explorou a API REST desprotegida do Trello, que permitia consultas não autenticadas por endereço de e-mail. Ao enviar endereços de e-mail de bancos de dados previamente violados, o invasor raspou mais de 15 milhões de perfis de usuários (e-mails, nomes completos, nomes de usuário, informações do fórum) e os publicou em um fórum de hackers. Posteriormente, a Atlassian restringiu o endpoint.

Fonte: BleepingComputer

High Julho de 2025 Vazamento de chave de API

Funcionário do DOGE publica chave de API privada xAI no GitHub - acesso a mais de 52 LLMs internos expostos

Um funcionário do Departamento de Eficiência Governamental (DOGE) dos EUA enviou um script (agent.py) contendo a chave de API privada do xAI para um repositório público do GitHub. A chave fornecia acesso irrestrito a mais de 52 modelos LLM, incluindo “grok 4-0709”, que havia sido criado apenas quatro dias antes. O GitGuardian detectou o vazamento, mas a chave não foi revogada imediatamente. Um vazamento semelhante também ocorreu em maio de 2025.

Fonte: Krebs sobre segurança

Active Exploitation Março de 2025 (exploração ativa observada) SSRF

CVE-2024-27564 - Mais de 10.000 tentativas de ataque na vulnerabilidade SSRF do ChatGPT

Uma vulnerabilidade de Server-Side Request Forgery (SSRF) no componente pictureproxy.php do ChatGPT. Um invasor não autenticado pode injetar URLs criados para forçar o servidor a executar solicitações internas não autorizadas. Mais de 10.000 tentativas de ataque foram registradas a partir de um único endereço IP, com 33% dos alvos sendo organizações dos EUA (setores financeiro, governamental e de saúde). A vulnerabilidade é particularmente perigosa porque pode ser explorada em grande escala sem autenticação.

Fonte: SecurityWeek

Lição aprendida: SSRF é uma ameaça séria às APIs

CVE-2024-27564 é um exemplo clássico de OWASP API7 (SSRF). APIs que aceitam entrada de URL de fontes externas exigem validação rigorosa para impedir o acesso a redes internas. Veja OWASP Top 10 → API7.

📊 Relatórios e tendências do setor

CVEs relacionados à IA aumentam 1.025% ano após ano

De acordo com o relatório API ThreatStats de 2025 da Wallarm, 439 CVEs relacionados à IA foram registrados em 2024, um aumento de 1.025% ano após ano. Mais de 50% das entradas do catálogo de vulnerabilidades exploradas conhecidas (KEV) da CISA estão relacionadas à API. As falhas de controle de acesso aumentaram 40% no geral e as falhas críticas de autorização aumentaram 36%.

99% das organizações enfrentaram problemas de segurança de API

De acordo com o relatório do primeiro trimestre de 2025 da Salt Security, 99% das organizações pesquisadas enfrentaram pelo menos um problema de segurança de API nos últimos 12 meses. Os principais riscos foram ataques de injeção e BOLA (autorização em nível de objeto quebrado), responsáveis ​​por mais de um terço de todos os incidentes. 95% dos ataques à API originaram-se de sessões autenticadas.

Salt Security Q1 2025

30.000 espaços de trabalho do Postman encontrados expondo chaves de API

Pesquisadores de segurança descobriram 30.000 espaços de trabalho do Postman expostos publicamente sem controles de segurança. Eles continham chaves de API ativas, tokens de acesso e cargas confidenciais, muitos deles fornecendo acesso direto a ambientes de produção. Este caso destaca o risco de credenciais de “API sombra” em ferramentas de colaboração de desenvolvedores.

📈 Tendências de segurança de API para 2024-2025

1. Rápido aumento nas vulnerabilidades da API AI/LLM

Vulnerabilidades em APIs baseadas em IA, como ChatGPT e DeepSeek, estão aumentando. Injeção imediata, SSRF e exposição de dados são as principais ameaças. As medidas de segurança não acompanham a rápida adoção da IA.

2. Autorização quebrada continua sendo a principal ameaça

BOLA (Broken Object Level Authorization) continua a ser a vulnerabilidade mais prevalente. Como visto nos casos Dell e Trello, a falta de verificações básicas de autorização leva a violações de dados em grande escala.

3. Ataques de sessões autenticadas

95% dos ataques ocorrem após a autenticação ter sido aprovada. Os invasores com tokens de acesso legítimos exploram falhas de autorização para se moverem lateralmente, tornando esta a técnica de ataque dominante.

4. Vazamentos de credenciais por meio de ferramentas de desenvolvedor

Vazamentos de chaves de API de ferramentas de desenvolvedor como GitHub e Postman se tornaram um problema crítico. A adoção de ferramentas de detecção automatizadas (por exemplo, GitGuardian) e práticas rigorosas de gerenciamento de segredos são essenciais.

✅ Ações a serem tomadas agora

Contramedidas aprendidas com esses incidentes
  • Aplicar verificações de autorização - Implementar autorização em nível de objeto em todos os endpoints da API (lição da Dell/Trello)
  • Implementar limitação de taxa - Defina limites de taxa apropriados por endpoint (a Dell não tinha limites, permitindo 5.000 solicitações por minuto)
  • Automatize a verificação de segredos - Detecte automaticamente vazamentos de credenciais de repositórios/espaços de trabalho públicos no GitHub, Postman, etc.
  • Implementar contramedidas SSRF - Valide URLs fornecidos pelo usuário e bloqueie intervalos de endereços IP privados
  • Auditar configurações de exposição do banco de dados - Verifique regularmente se as interfaces de gerenciamento não são expostas acidentalmente (lição do DeepSeek)
  • Avaliação de segurança para endpoints de IA/LLM - Realize avaliações de risco para injeção imediata e vazamento de dados
  • Verificações de vulnerabilidade de dependência - Integre auditoria npm/Snyk em CI/CD para detectar automaticamente CVEs conhecidos
